- **The demo seed is now FORCE-ROW-LEVEL-SECURITY-native — it works on a production-shaped DB without weakening RLS.** `seed/bootcamp_curriculum.py` set only the `use_org` ContextVar (app-layer write-stamp) and never pushed the tenant to Postgres' `app.current_org` GUC, so under prod `FORCE ROW LEVEL SECURITY` on a non-superuser role every org-scoped INSERT was rejected (`new row violates row-level security policy for table "curricula"`). The whole test suite runs as a superuser (RLS-bypassed), which is exactly why the gap shipped. Fix: a new `app/database.py::bind_session_org(session, org_id)` helper pushes the org to the DB GUC at **session scope** (`set_config('app.current_org', :org, false)`), so it survives the flushes and mid-block commits inside a tenant block AND updates when the seed switches orgs — including several orgs within a **single transaction** (`seed()`'s one commit), where a transaction-local `SET LOCAL` GUC would go stale on the switch. A seed-local `_bound_org(session, org_id)` async context manager enters `use_org` **and** binds the GUC together, and replaces every bare `with use_org(...)` at the seed's tenant blocks (`_seed_org`, `_demo_enrich`, `_demo_review_ccr`, `_demo_learner_enroll`); `app/migration/backfill_content_model.py` (run on the same session by the seed CLI) binds the GUC per org too. **No migration**, no change to `get_db`/`app.tenant`/RLS policies — the value is always bound, never interpolated (no injection surface), and the extra `set_config` is a no-op under an RLS-bypassing dev/CI role, so existing behavior is unchanged. Proven by reproducing FORCE-RLS locally (non-superuser role subject to the forced policy): red = the `curricula` RLS rejection above; green = a full multi-org seed (both orgs, curricula, back-fill, demo v1.1.0 releases, review CCRs, learner enrollment) completing successfully. New regression test `tests/integration/test_tenant_isolation.py::test_bind_session_org_admits_writes_and_survives_org_switch` drops to a throwaway NOSUPERUSER role and asserts a bound write succeeds, survives an org switch in one transaction, and is rejected (SQLSTATE 42501) once the bind is removed. `scripts/seed_prod.py`'s temporary un-force-RLS step is now optional belt-and-suspenders rather than required.
